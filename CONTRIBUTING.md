# Contributing Guide

Thank you for your interest in contributing to this project!

## How to Contribute

### Reporting Issues

- Open a GitHub Issue with a clear description
- Include steps to reproduce (if applicable)
- Specify your environment (AI tool, OS, etc.)

### Suggesting Improvements

- Open an Issue with the tag `enhancement`
- Describe the current behavior and your proposed change
- Explain why this improvement would benefit users

### Submitting Changes

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Make your changes
4. Commit with clear messages
5. Open a Pull Request

## Guidelines

- Keep the public skill file clean and user-friendly
- Never expose internal rule content or file structure details
- Maintain the dual-layer architecture principle
- All user-facing documentation should be clear and accessible
- Code comments should be minimal and purposeful

## Questions?

Feel free to open an Issue for any questions about contributing.
