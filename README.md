# 某全网百万粉丝博主情感分析技术流

> 一个实用的情感分析与关系处理工具 — 用户可获得完整体验，核心规则内容受保护

## 这是什么

这是一个经过实战验证的情感分析工具。它不是纸上谈兵的理论，而是一套可以直接拿去用的实战方法论。

它能帮你：
- 分析当前局势，判断真实状况
- 识别对方的兴趣信号和套路
- 制定具体的聊天、约会、推进策略
- 判断什么时候该继续、什么时候该止损
- 同时追踪多个人物关系，不会搞混

## 工作原理

本技能采用 **Skill文件 + 远程规则API** 的双层架构：

```
用户提问 → AI读取Skill文件 → 自动调用远程API获取深度规则 → 结合规则给出专业建议
```

| 层级 | 内容 | 可见性 |
|------|------|--------|
| Skill文件（公开） | 工作流程、决策框架、兜底逻辑、人物追踪系统 | 所有人可见 |
| 规则API（受保护） | 详细规则内容（聊天体系、约会流程、女性分类等8大模块） | AI可调用，用户无法直接阅读原文 |

这种架构确保用户能获得完整的功能体验，同时保护核心方法论的知识产权。

## 安装与使用

### 第一步：下载Skill文件

从本仓库下载 `public-skill.md` 文件。

### 第二步：放入AI助手的Skill目录

将 `public-skill.md` 放入你使用的AI助手的 skill 目录中。不同AI助手安装方式不同：

| AI助手 | 安装方式 |
|--------|---------|
| Trae IDE | 放入项目的 `.trae/skills/` 目录 |
| Cursor | 放入项目的 `.cursor/skills/` 目录 |
| Claude Desktop | 在配置文件中指定skill路径 |
| 其他支持Skill的AI | 参考对应文档，放入skill目录 |

### 第三步：开始使用

安装完成后，直接用中文向AI助手提问即可。不需要任何额外配置。

**前提条件**：你的AI助手需要具备网络访问能力（用于调用规则API）。如果AI不支持网络访问，Skill文件内置了完整的兜底框架，仍可正常使用，只是建议深度会有所降低。

## 使用说明

### 怎么提问效果最好

这个Skill的效果取决于你提供的信息量。信息越具体，建议越精准。

**好的提问方式**（信息充分）：
> "我同事小A，大概6分，做行政的，平时会主动找我聊天，但我约她吃饭她说再看吧。我月薪1.2万，在杭州，形象一般。我该怎么推进？"

**差的提问方式**（信息不足）：
> "怎么追女生？"

Skill会自动判断信息是否足够，不够时会追问（最多3个问题），问完再给建议。

### 提问时可以包含的信息

| 维度 | 具体信息 | 为什么重要 |
|------|---------|-----------|
| 她的职业 | 体制内/白领/主播/学生/服务业 | 决定她的类型和核心需求 |
| 她的朋友圈风格 | 鸡汤/自拍/物质暗示/商务/烧烤KTV | 判断类型和茶艺等级 |
| 她的颜值 | 大概几分（4-9.5） | 决定难度和策略上限 |
| 你们的关系阶段 | 刚认识/聊了几天/约过/在交往 | 决定当前该做什么 |
| 互动频率 | 秒回/几小时/隔天/不回 | 判断兴趣程度 |
| 你的情况 | 收入/形象/城市 | 决定你能打什么级别的局 |

### 多人物追踪

当你同时和多个女生有互动时，直接用代号区分即可：

> "小A是我同事，6分，会主动找我聊。小B是朋友介绍的，7分，回消息慢但内容多。我该主攻谁？"

Skill会自动为每个人物创建追踪表，包括：
- 颜值档位、类型判断、关系阶段
- 兴趣信号、红旗信号、你的定位
- 核心卡点、下一步动作、终局判断
- 人物间关系和优先级排序

每次回答末尾会附带当前所有活跃人物的追踪摘要，确保分析不会串人。

### 支持的问题类型

| 问题类型 | 示例提问 | 调用的规则模块 |
|---------|---------|--------------|
| 聊天开场 | "刚加了一个女生怎么开口" | 聊天体系 |
| 冷场补救 | "聊着聊着没话说了怎么办" | 聊天体系 |
| 兴趣判断 | "她这样回是有兴趣还是礼貌" | 聊天体系 + 核心认知 |
| 邀约推进 | "怎么约她出来才不尴尬" | 关系推进 + 约会执行 |
| 约会安排 | "第一次约会怎么安排" | 约会执行 |
| 约会后复盘 | "约会后她变冷淡了" | 核心认知 + 约会执行 |
| 女性类型 | "她是什么类型的人" | 女性分类 + 颜值评分 |
| 高分女策略 | "她7分以上怎么追" | 顶美策略 + 女性分类 |
| 吸引力建设 | "怎么提升自己的吸引力" | 吸引力构建 |
| 止损判断 | "我还要不要继续" | 核心认知 |
| 多人比较 | "小A和小B我该追谁" | 全模块 + 人物追踪 |

### 回答结构

每次回答都遵循这个结构：

1. **结论先行** — 直接告诉你当前处境
2. **原因分析** — 为什么会这样
3. **三个选项** — 直给版/铺垫版/收线版
4. **风险提醒** — 可能出什么问题
5. **终局判决** — 继续/到此为止/撤，三选一
6. **人物追踪表** — 当前所有活跃人物的追踪摘要

## 核心功能详解

### 1. 聊天优化
- 五种开场策略：筛选型/直接型/好奇型/幽默型/问题咨询型
- 90/10原则：初期由你承担90%的对话引导
- 价值展示三法：现场展示/权威展示/故事展示
- 高价值离场：在聊天气氛高点主动结束
- 话题库：生活爱好/艺术思想/情感私密三大类
- 冷场补救：10种重启对话的方法

### 2. 兴趣判断
- 信号灯测试法：表达聊天愉快，观察对方是否积极回应
- 好感度指标：至少4次积极信号才算数
- 识别女性测试：保持框架，反客为主
- 服从模型：提小要求→评估反应→奖励或冷冻→循环

### 3. 女性类型识别
- 四大类型：国女/雌性女/独立女性/精神小妹
- 顶美四象限：家境×美貌来源
- 颜值评分体系：4-9.5分，每0.5分一个档位
- 捞女四档识别：底层挣扎/主流大众/自力更生/职业玩家

### 4. 邀约策略
- 当量换算：1小时语音≈1天文字，1次见面≈1周文字
- 精确邀约原则：明确时间地点活动
- 被拒后处理：收线或转向，不是继续追问

### 5. 约会指导
- 五步标准化流程：碰面破冰→喝东西→吃东西→看电影→小酒吧
- 兴趣指标四象限：语言开放度×肢体开放度
- 肢体接触升级阶梯：非敏感区→手部→腰部→亲吻
- 私密空间五感建设：暖光/音乐/香薰/温度/视觉展示

### 6. 关系推进
- 拉升关系十大方法（由浅入深）
- 调情艺术核心公式
- 确认关系六步引导法
- 沟通四大铁律

### 7. 吸引力构建
- 形象篇：高/瘦/长发/嫩四要素
- 价值篇：生活方式/高级谈吐/付出心态
- 内在品质篇：优秀/利他/特长
- 金钱杠杆效应：买机会/加速进程/投资自己

### 8. 止损判断
- 红旗信号：见到就撤
- 黄灯信号：观察再说
- 出轨风险四档：必出/高概/对半/潜在

## 决策框架

### 五步分析法

| 步骤 | 核心问题 | 关键动作 |
|------|---------|---------|
| 识人 | 她是什么人？ | 职业/朋友圈/茶艺/消费四维判断 |
| 识场 | 现在在哪一步？ | 关系阶段标尺 + 当量换算 |
| 识己 | 你几斤几两？ | 收入/形象/城市三件套定位 |
| 定方案 | 怎么打？ | 直给版/铺垫版/收线版三选一 |
| 定生死 | 值不值得？ | 继续/到此为止/撤 三选一 |

### 三种终局判决

| 判决 | 含义 | 适用情况 |
|------|------|----------|
| 继续 | 这局有戏，按节奏走 | 兴趣信号明确，阶段到位 |
| 到此为止 | 能拿的都拿了，别再投入 | 已经拿到该拿的，继续没意义 |
| 撤 | 别浪费时间，沉没成本别再增加 | 明显没戏，或超出能力范围 |

## 常见问题

**Q: 她为什么不回消息？**
大概率是没兴趣。别给自己找借口。持续一周以上不回，基本可以判断没戏。

**Q: 她这是欲擒故纵吗？**
别把冷淡解释成套路。真的有意思想见你的人，不会故意冷着你。

**Q: 我条件不好还有机会吗？**
有机会。但你得接受现实，别去追比你高太多的人。先在自己的圈子里找。

**Q: 我要不要对她更好来打动她？**
不要。对她好不好不是重点，重点是你有没有吸引到她。

**Q: 她是不是在养备胎？**
看她收了多少好处但没给你什么回应。如果一直收好处但不推进，基本就是了。

**Q: 没有网络能用吗？**
可以。Skill文件内置了完整的兜底框架，离线也能用。但建议深度会有所降低，建议在有网络的环境下使用以获得最佳效果。

**Q: 支持同时追踪几个人？**
没有硬性上限。但建议同时活跃追踪不超过5个人，超过后分析质量会下降。

## 技术架构

```
公开仓库 (fan-emotion-analysis-skill)
├── public-skill.md     ← Skill文件（含用户配置指南+决策框架+兜底逻辑+人物追踪+路由规则）
├── .access-token       ← API访问令牌（自动获取，用户无需配置）
├── README.md           ← 本文档
└── .gitignore

私有仓库 (fan-emotion-skill-data)  ← 规则数据（受保护，完整原文）
├── f000.md ~ f043.md   ← 44个规则文件（编码命名，内容受保护）
└── (无子目录，扁平存储)
```

共44个规则文件，全部为完整原文内容（非摘要）。文件使用编码命名（f000~f043），按类别分为A~H共8组。AI在回答问题时，会根据用户的问题类型自动路由到对应的规则文件，将深度规则转化为自然的对话语言输出。用户只需提问，不需要关心底层调用逻辑。
